# Community Call

The Community Call is a quarterly, public, recorded meeting where maintainers share what shipped, what's in flight, and where the project is going — and where the community can ask questions, surface concerns, and showcase work built on top of the engine.

This directory contains the operating documents for the call: cadence, templates, runbooks, and tooling notes. The actual calls begin once there's an active community to call together; until then, this is the playbook.

## Cadence

- 4 calls per year, one near the end of each quarter (Q1, Q2, Q3, Q4)
- 60 minutes per call
- Dates announced at least 4 weeks ahead
- Each call is anchored to a fixed weekday/time slot (e.g., last Thursday of the quarter, 16:00 UTC) so it's predictable across timezones; concrete slot is set in the [per-call document](./2026-Q3-call.md) and the [communication channels](./communication-channels.md)

## Format

A standard agenda — see [`agenda-template.md`](./agenda-template.md) — frames every call:

1. 5min welcome + this-quarter recap
2. 25min product update from maintainers (what shipped, what's in flight)
3. 15min community spotlight (one contributor presents)
4. 10min Q&A from submitted + live questions
5. 5min wrap + announce next call's date

The spotlight slot is sourced from GitHub Discussions — see the facilitator runbook for how to recruit.

## Attendance

- **Open to anyone.** No NDA, no sales gate.
- Registration via a public form (see [`recording-setup.md`](./recording-setup.md) for the form choice).
- Joining details are emailed on registration and posted in the channels listed in [`communication-channels.md`](./communication-channels.md) ~24h before.
- Attendees join through the link in [`attendee-runbook.md`](./attendee-runbook.md).

## Recordings

- Every call is recorded with the speaker's consent (acknowledged at registration and at the top of the call).
- Recordings + auto-generated transcripts are uploaded to YouTube within 5 business days.
- Each recording is linked from `docs.bzonfhir.com/community-calls` and from the per-call document in this directory.
- See [`recording-setup.md`](./recording-setup.md) for the toolchain.
