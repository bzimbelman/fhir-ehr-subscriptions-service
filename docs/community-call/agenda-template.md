## Agenda

### 1. Welcome + this-quarter recap — 5 minutes

- Welcome attendees, remind them the call is being recorded
- Confirm code of conduct ([`CODE_OF_CONDUCT.md`](../../CODE_OF_CONDUCT.md)) applies
- One-slide recap: top 3 things that landed since the last call
- Quick attendance / introductions in chat (no round-robin — keeps time)

### 2. Product update from maintainers — 25 minutes

What shipped, what's in flight, what's next. Suggested structure:

- 10min — Shipped this quarter (link each item to its PR / release notes)
- 10min — In flight (roadmap items currently being worked, expected landing window)
- 5min — Next quarter focus + areas where help is wanted

Maintainers should pre-record demos longer than 2 minutes and embed them rather than live-demoing.

### 3. Community spotlight — 15 minutes

One community contributor presents something they've built, integrated, or learned. Sourced from GitHub Discussions — see the facilitator runbook for how to recruit.

- 10min — Presentation
- 5min — Q&A with the presenter

### 4. Q&A — 10 minutes

- Submitted questions first (collected via the registration form + Discussions thread)
- Live questions from chat / "raise hand" if time allows
- Anything unanswered moves to a follow-up Discussions thread within 5 business days

### 5. Wrap + next call — 5 minutes

- Recap action items from the call (notetaker reads them back)
- Announce the next call's date and where the recording / transcript will appear
- Thank the spotlight presenter
