# Trademark Filing Playbook — `bzonfhir` Wordmark

This document captures the USPTO trademark filing process for the **bzonfhir**
wordmark, as called for by §8.1 of the master plan. The actual filing is a
manual action for Brian (or his attorney). This playbook is the prep work:
the decisions, the application data, the timeline, and the checklist.

## TL;DR

- File **bzonfhir** as a **wordmark** (not a logo) in **International Class 042**
  (computer software / SaaS).
- Use **TEAS Standard** (not TEAS Plus) because the goods/services description
  is custom.
- File as **Brian Zimbelman, individual**, and assign the registration to the
  LLC after incorporation.
- **Date of first use in commerce: 2026-06-27** — the day bzonfhir.com went
  live with the wordmark visible in the TopNav.
- Expected cost: **$250 – $350** USPTO filing fee per class, plus **$500 –
  $2,000** if an attorney is used.
- Expected timeline: **3–4 months to publication**, **30-day opposition
  window**, **registration 4–6 months after publication**.

## 1. The decision: wordmark, not logo

A **wordmark** protects the text "bzonfhir" in any font, size, color, or
styling. A **logo mark** would protect a specific stylized rendering of the
text or an associated graphic.

We file the **wordmark** because:

- The brand value is in the name itself — "bzonfhir" is a coined term that
  combines the user's initial with "on FHIR."
- A wordmark gives broader protection: nobody can use "bzonfhir" in any visual
  form for competing IC 042 services, regardless of how it is styled.
- The TopNav rendering may evolve. We do not want the trademark tied to a
  specific font or color.
- If the brand later acquires a distinctive logo, that logo can be filed
  separately as a design mark without disturbing the wordmark.

## 2. International Class — IC 042

**IC 042** (Computer & Scientific Services) is the standard class for
software-as-a-service offerings. It covers:

- Software-as-a-service (SaaS)
- Platform-as-a-service (PaaS)
- Hosting / cloud computing
- Software design, development, and maintenance services
- Downloadable software in a SaaS context

We file in **IC 042 only** for the initial application. We can add other
classes later (e.g., IC 009 for downloadable software products, IC 041 for
educational services) if the business expands. Each additional class is an
additional filing fee.

## 3. Goods and services description (verbatim)

This is the text the application will submit to the USPTO:

> Software-as-a-service (SAAS) services featuring software for the integration
> of healthcare data and electronic health records using FHIR and HL7
> standards; downloadable computer software for the integration of healthcare
> data using FHIR and HL7 standards

This description is **custom** — it is not pulled verbatim from the USPTO's
pre-approved Trademark ID Manual. That is why we use **TEAS Standard** rather
than TEAS Plus.

## 4. TEAS Standard vs TEAS Plus

The USPTO offers two electronic filing tiers:

| Feature | TEAS Plus | TEAS Standard |
| --- | --- | --- |
| Fee per class (as of 2026) | $250 | $350 |
| Goods/services description | Must use exact wording from the USPTO ID Manual | Custom wording allowed |
| Additional requirements | All info upfront (specimen, declaration, etc.) | Allowed to defer some items |
| Office-action risk | Low (because pre-approved) | Higher (custom wording invites scrutiny) |

We choose **TEAS Standard** because our description includes "FHIR" and "HL7"
— industry-specific terms that do not appear in the USPTO's pre-approved list.
Trying to force-fit a pre-approved description would either narrow our
protection or describe services we do not offer.

The extra **$100/class** is worth it to keep the description accurate.

## 5. Specimen of use in commerce

A "specimen" is evidence that the mark is actively used in commerce for the
services claimed. For SaaS, the standard specimen is a **screenshot of the
live website** showing the mark in the context of the services.

The specimen for this filing will be:

- A screenshot of **bzonfhir.com** (the production site) captured on or after
  2026-06-27.
- The screenshot must clearly show:
  - The **"bzonfhir"** wordmark in the TopNav.
  - Context establishing that the services described (FHIR/HL7 integration
    SaaS) are actually offered — e.g., the hero text, a feature panel, or a
    "Get Started" call-to-action.
- File the screenshot as a PNG or JPG at the resolution captured. Do not
  scale it down — the examining attorney needs to be able to read the
  wordmark and the surrounding context.

## 6. Filing identity — Brian, individually

The application will be filed in the name of **Brian Zimbelman, individual**,
because:

- The LLC has not yet been incorporated (that is a separate work item, see
  §8.2 of the master plan).
- Filing as an individual gets the **priority date** locked in immediately.
  Waiting for incorporation delays the priority date.
- After the LLC is incorporated, the trademark can be **assigned** to the LLC
  via a simple USPTO assignment filing (~$40, a few minutes).
- Filing in the individual name first and assigning later is a standard
  pattern; it does not weaken the registration.

## 7. Date of first use in commerce

**Date of first use anywhere:** 2026-06-27
**Date of first use in commerce:** 2026-06-27

Both dates are the same: the day **bzonfhir.com** went live in production
with the **bzonfhir** wordmark visible in the TopNav. "Use in commerce" for
SaaS means the service is publicly available and the mark is displayed in
connection with offering the service — which the live website satisfies.

If pressed for earlier dates (e.g., development screenshots, internal demos),
do not claim them. Those are not "use in commerce." Stick with the public
launch date.

## 9. Timeline expectations

USPTO trademark processing is slow. Plan for the long road.

| Milestone | Expected timing |
| --- | --- |
| File application | Day 0 |
| Application receipt + serial number | Within 1 week |
| Initial examining attorney review | 3–4 months after filing |
| Office Action (if any) — response due | 3 months from action |
| Publication for opposition | 3–4 months after filing (if no office action) |
| Opposition window | 30 days from publication |
| Notice of Allowance / Registration | 4–6 months after publication |
| **Total elapsed time to registration** | **~9–12 months** |

## 10. What happens during the wait

- **Keep using the mark.** Continued use in commerce strengthens the
  application.
- **The application establishes a priority date** of the filing day. Anyone
  who starts using "bzonfhir" after that date is junior to our claim, even
  before registration completes.
- **Watch for office actions.** The USPTO will email if the examining
  attorney has questions or refusals. Respond within 3 months or the
  application is abandoned.
- **Watch the opposition window.** If a third party opposes, it triggers a
  TTAB proceeding. Rare for coined terms, but possible.
- **Do not use the ® symbol yet.** Until registration is granted, use **™**
  to indicate a claimed trademark. Switch to ® only after the certificate is
  issued.

## 11. Defensive posture

Three different legal regimes are in play; do not confuse them:

- **Trademark** protects the **word "bzonfhir"** as a brand identifier for
  the services described. It does not cover the code.
- **Copyright** protects the **source code** automatically from the moment
  of authorship. No filing required, but registering with the U.S. Copyright
  Office is cheap and useful for litigation. Separate work item.
- **Apache License 2.0** is the license under which the code is published.
  It grants permissive use of the code but **does not grant any trademark
  rights** — Apache 2.0 §6 explicitly excludes trademarks. Anyone may fork
  the code under Apache 2.0; nobody may call their fork "bzonfhir."

Treat these three as orthogonal. The trademark filing is purely about the
name.
